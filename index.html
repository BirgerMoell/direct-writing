Codex Skill

Write About The Subject, Not The Container.

Direct Writing removes copy that talks about the page, course, repo, example, author, agent, or writing process when the real subject is the method, artifact, claim, or action.

The Problem

Self-reference weakens technical copy.

Phrases like "this page explains", "students can study", and "the repo includes" move attention away from the actual concept. The skill rewrites those sentences so the method or artifact carries the claim.

It does not ban words like course, student, page, or repository. It keeps them when they are the real subject, such as deadlines, installation, policy, or publishing details.

Rewrite Rules

Four moves do most of the work.

  1. Name the real subject.DPO, GRPO, local inference, the checkpoint, the evaluator, the reader action.
  2. Replace meta verbs.Turn "shows", "teaches", and "explains" into "compares", "scores", "loads", "verifies", or "trains".
  3. Keep operational facts.Commands, deadlines, paths, assignment policies, and install steps stay concrete.
  4. Run the portability test.If a sentence fails outside the current page, rewrite it around the concept.

Install

Add It To Codex.

Clone the repository into the Codex skills directory. Use CODEX_HOME if it is set; otherwise use ~/.codex.

Use

Trigger it whenever prose drifts into meta commentary.

In Codex, invoke it explicitly with $direct-writing, or let it trigger when a writing task asks for less self-referential, less meta, more direct, or more subject-centered copy.

In other tools, use the repository's SKILL.md as a project rule, agent instruction, or reusable editing prompt. The pattern library lives in references/patterns.md.
